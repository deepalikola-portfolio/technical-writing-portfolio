# Kelvyn Park Chat Assistant — Documentation Oversight Case Study

## Public Repository
https://github.com/ASUCICREPO/kelvyn-park-chat-assistant

## Project Context
The Kelvyn Park Chat Assistant is an AI-powered conversational system developed through the ASU Cloud Innovation Center to support school stakeholders with automated responses and information access.

The system integrates conversational AI with structured response workflows to assist students, staff, and administrators.

## My Role
I did not author the original closure documentation but oversaw documentation quality and edited technical content to improve clarity, structure, and usability.

I also used the documentation to create marketing and communication assets that explained the system’s functionality to stakeholders.

## Responsibilities
- Reviewed technical documentation written by project engineers
- Edited documentation for readability and logical structure
- Ensured system workflows were clearly explained
- Translated technical descriptions into stakeholder-facing messaging

## Documentation Focus

Key areas I reviewed and edited included:

- System overview and purpose
- Chatbot interaction flow
- AI response handling logic
- Terminology clarity for non-technical audiences

## Outcome
Improved documentation clarity and supported the creation of accurate marketing assets that communicated the system’s capabilities.

## Note
This case study references a public repository and does not include internal documentation.
