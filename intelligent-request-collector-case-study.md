# Intelligent Request Collector — Documentation Oversight Case Study

## Public Repository
https://github.com/ASUCICREPO/intelligent-request-collector

## Project Context
An AI-powered request intake and routing system developed through the ASU Cloud Innovation Center in collaboration with the Chandler Police Department.

## My Role
I did not author the original closure documentation but oversaw documentation quality and edited technical content for clarity, structure, and usability.

I also translated the documentation into marketing-facing assets used for stakeholder communication.

## Responsibilities
- Reviewed documentation produced by engineering teams
- Edited sections to improve clarity and structure
- Ensured documentation hierarchy made the system understandable
- Converted technical information into marketing messaging and assets

## Documentation Focus
Key areas I reviewed and edited included:

- System overview explanation
- Workflow logic
- Request processing flow
- Technical terminology clarity

## Outcome
Improved documentation readability and enabled accurate marketing assets aligned with the system’s real functionality.

## Note
This case study references a public repository and does not include internal project documentation.
